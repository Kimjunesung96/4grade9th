프로젝트 기획 배경 Problem & Solution

기존의 한계 (Problem)

  • ▪ 전문 설계 CAD 툴은 진입 장벽이 너무 높음
  • ▪ 건설 샌드박스 게임은 객관적인 수치 데이터를 제공하지 않음

프로젝트 목표 (Solution)

"무너지기 전에 미리 알고, 대비할 수는 없을까?"

  • 1. 직관적이고 쉬운 건설 환경 구축
  • 2. 실시간 물리 스트레스 정밀 진단
  • 3. 데이터 기반 재료 추천 및 보강 솔루션 제공

핵심 기술: 대용량 데이터 파이프라인

보강 도면 생성 로직 (ReinforcementManager.cs)

// 위험 블록 하단으로 파고들며 보강 타설
while (currentY >= 45) {
    currentY -= 30;
    // 1. 하단 구조물 충돌 방지(HashSet 확인)
    if (existingBlocks.Contains(targetId)) break;
    existingBlocks.Add(targetId);
    // 2. 도면 리스트 등록
    planLines.Add($"{targetId},{cleanX},{targetY},{cleanZ},Reinforcement");
}

다중 스레드 병렬 처리를 통해 수만 개의 블록 연산을 효율화했습니다. ReinforcementManager의 2패스 알고리즘을 통해 실제 물리 보강 타설 좌표를 CSV로 자동 추출합니다.

도면 ➜ 엑셀(CSV) 다이렉트 변환

사용자가 넣은 설계도(이미지)를 즉시 스캔하여, 3D 물리 연산이 가능한 엑셀(CSV) 데이터로 자동 변환하는 통합 제어 파이프라인 구축

R&D: 보강 알고리즘과의 사투 Problem → Solution

문제 상황

  • ▪보강 로직을 AI에게 맡겼으나 정교한 설계도 생성 실패
  • ▪수만 개 블록 좌표 추출 요청 시 토큰 한계로 출력 중단
  • ▪2주간의 시도 끝에 외부 AI 의존 방식 전면 포기

직접 설계한 해결책

  • ▪위험 하중 분포를 시스템이 스스로 구분하는 로직 설계
  • ▪허공에서 바닥을 향해 기둥을 뻗어 내리는 기초 기둥 보강 알고리즘 고안
  • ▪코드 구현은 AI 보조, 핵심 논리는 직접 설계하여 완성
